## PRD: Frontend-архитектура Календаря Мероприятий

### 1\. Обзор приложения и Стек технологий

Frontend-приложение отвечает за рендеринг публичного расписания для зрителей и предоставление панели управления для администраторов.

- **UI-библиотека:** shadcn/ui (Radix UI под капотом)
- **Маршрутизация:** TanstackRouter v6
- **Валидация:** Zod (использование общих схем с бэкендом)
- **Управление серверным состоянием (API):** TanStack Query

### 3\. Пользовательский Интерфейс (UI/UX)

#### 3.1. Публичный Календарь (/)

- **Макет:** На всю ширину экрана с шапкой (Логотип, Месяц/Год, кнопка "Войти" для админа).
- **Сетка Месяца:**
  - Кастомный компонент на базе CSS Grid (7 колонок — дни недели).
  - В ячейках дней отображаются компактные блоки ("чипсы") мероприятий.
  - Цвет фона "чипса" берется из связанного EventType.color (через inline-style).
  - При наведении на событие (Hover) легкое изменение прозрачности или тени.

- **Просмотр мероприятия (Модалка/Drawer):**
  - При клике на "чипс" открывается компонент Dialog или Sheet (справа).
  - **Содержимое:** Название мероприятия, бейдж с типом мероприятия, локация (название + адрес), точное время начала и конца, описание (если есть), список задействованных сотрудников (сгруппированный по ролям, если позволяет API).
  - _Примечание:_ Фильтры в публичном виде отсутствуют согласно требованиям.

#### 3.2. Панель Администратора (Защищенная зона)

- **Навигация (Sidebar / Navbar):** Переключение между календарем, локациями, типами мероприятий и сотрудниками. Кнопка выхода (Logout).
- Административный Календарь (/admin/calendar):
  - Визуально идентичен публичному, но с интерактивностью.
  - **Создание:** Клик по пустому месту в ячейке дня открывает модальное окно создания мероприятия с предзаполненной датой.
  - **Редактирование/Удаление:** Клик по существующему событию открывает форму редактирования с кнопкой "Удалить событие" (красная, variant="destructive").

### 4\. Требования к Формам и Валидации

Все формы в панели администратора должны строиться на react-hook-form и валидироваться через общие Zod-схемы.

**4.1. Форма Создания/Редактирования мероприятия (EventForm)**

- Компоненты поля shadcn (FieldGroup, Field, FormLabel, etc…) используются для автоматического отображения ошибок, например, если дата окончания меньше или равна дате начала (endAt <= startAt).
- **Поля:**
  - title: Input (текст).
  - description: Textarea (опционально).
  - typeId: Select (данные загружаются из /api/admin/event-types).
  - locationId: Select (данные загружаются из /api/admin/locations).
  - startAt / endAt: Комбинация Popover, Calendar (из shadcn) для выбора даты и нативного или кастомного селекта для выбора времени.
  - employeeIds: Компонент Multi-Select (можно использовать Command / Popover из shadcn для создания выпадающего списка с чекбоксами).

- **Обработка ошибок:** Если сервер возвращает статус 409 Conflict (пересечение расписания), сообщение об ошибке должно выводиться поверх формы через глобальный Toast.

**4.2. Формы Справочников**

Для каждой сущности (Локация, Тип мероприятия, Сотрудник) реализуются компактные формы в модальных окнах (Dialog).

- **Тип мероприятия:** Поле color должно представлять собой визуальный Color Picker (или нативный ), который возвращает HEX-строку для валидации.

### 5\. Управление Состоянием и Данными (Tanstack Query)

Для оптимизации работы приложения и кэширования ответов от Hono бэкенда используется Tanstack **Query**.

- **Инвалидация кэша:**
  - При успешном создании/изменении/удалении мероприятия (мутация) необходимо вызывать queryClient.invalidateQueries({ queryKey: \['events'\] }), чтобы сетка календаря обновилась мгновенно.
  - При изменении справочников (например, переименовании локации) инвалидировать ключи \['locations'\] и \['events'\].

- **Оптимистичные обновления (Optimistic Updates):** (Опционально) Для улучшения UX при перетаскивании мероприятий (если будет реализовано) можно обновлять локальное состояние календаря до ответа от сервера, откатывая его при ошибке.
- **Загрузка данных по месяцам:** Календарь запрашивает мероприятия не разом, а передавая параметры start и end текущего просматриваемого месяца. При переключении месяца (стрелки "Вперед/Назад") Tanstack Query автоматически кэширует просмотренные месяцы.

### 6\. Компоненты UI (Спецификация shadcn/ui)

Перечень компонентов shadcn, которые необходимо инициализировать (npx shadcn-ui@latest add ):

- button (Кнопки действий)
- dialog (Окна редактирования и создания)
- sheet (Просмотр мероприятия на мобильных или десктопе)
- field, (Построение форм)
- select (Выбор единственного значения: локация, тип)
- popover (Сложные выпадающие списки, Multi-select для сотрудников)
- calendar (Выбор конкретной даты в формах)
- sonner (Уведомления об успешном сохранении или конфликтах расписания)
- table (Отображение списков справочников в админке)
- badge (Визуализация типов мероприятий)
