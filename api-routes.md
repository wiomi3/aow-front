# Документация API: Система управления календарем мероприятий

Все маршруты API имеют префикс `/api`.  
Интерактивная документация (Swagger) доступна по адресу `/swagger-ui`.  
OpenAPI спецификация (JSON) доступна по адресу `/openapi.json`.

---

## 1. Публичные маршруты

Эти маршруты доступны всем пользователям без авторизации.

### Получение событий

**GET** `/api/events`

Возвращает список событий, которые пересекаются с указанным временным интервалом.

- **Query-параметры:**
  - `start` (string, ISO Date): Начало периода (например, `2026-05-01`).
  - `end` (string, ISO Date): Конец периода (например, `2026-05-31`).
- **Успешный ответ (200 OK):**
  ```json
  [
    {
      "id": "uuid",
      "title": "Вечерний джаз",
      "description": "Живая музыка",
      "startAt": "2026-05-20T18:00:00Z",
      "endAt": "2026-05-20T20:00:00Z",
      "type": { "name": "Концерт", "color": "#FF5733" },
      "location": { "name": "Зал 1", "address": "ул. Мира, 1" },
      "employees": [...]
    }
  ]
  ```

## 2. Маршруты администратора (Защищенные)

Все маршруты с префиксом `/api/admin/*` требуют авторизации с ролью `admin`.  
В случае отсутствия прав возвращается ошибка `401 Unauthorized`.

### 2.1. Управление событиями

#### Создать событие

**POST** `/api/admin/events`

- **Тело запроса:**
  ```json
  {
    "title": "Название",
    "description": "Описание (опционально)",
    "startAt": "ISO Date",
    "endAt": "ISO Date",
    "typeId": "uuid",
    "locationId": "uuid (опционально)",
    "employeeIds": ["uuid", "uuid"]
  }
  ```
- **Ошибки:**
  - `409 Conflict`: Если один из сотрудников уже занят в другом событии в этот период.

#### Обновить событие

**PUT** `/api/admin/events/:id`

- **Тело запроса:** Аналогично POST.
- **Ошибки:**
  - `409 Conflict`: Проверка накладок (исключая само редактируемое событие).

#### Удалить событие

**DELETE** `/api/admin/events/:id`

### 2.2. Справочники (Dictionaries)

Для всех справочников поддерживаются стандартные CRUD операции:

- `GET /api/admin/{resource}` — Список всех записей.
- `POST /api/admin/{resource}` — Создание.
- `PUT /api/admin/{resource}/:id` — Обновление.
- `DELETE /api/admin/{resource}/:id` — Удаление.

#### Локации (`locations`)

Управление площадками проведения мероприятий.

- **Схема:** `{ "name": "string", "address": "string" }`

#### Типы событий (`event-types`)

Категории (Концерт, Репетиция и т.д.) и их цвета в календаре.

- **Схема:** `{ "name": "string", "color": "HEX string (#RRGGBB)" }`

#### Типы сотрудников (`employee-types`)

Категории персонала (Вокалист, Техник и т.д.).

- **Схема:** `{ "name": "string" }`

#### Сотрудники (`employees`)

Список персонала с привязкой к типу.

- **Схема:** `{ "name": "string", "employeeTypeId": "uuid" }`
